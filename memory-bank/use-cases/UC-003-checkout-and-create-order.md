# UC-003: Checkout And Create Order

## Goal

Покупатель завершает путь от заполненной корзины до созданного заказа внутри Telegram-бота и получает номер заказа для дальнейшей коммуникации.

## Primary Actor

Покупатель интернет-магазина.

## Trigger

Пользователь открывает корзину с товарами и выбирает действие оформления заказа.

## Preconditions

- Пользователь зарегистрирован в таблице `users`.
- У пользователя есть непустая корзина в PostgreSQL.
- Товары, уже добавленные в корзину, доступны для повторного чтения при формировании итоговой сводки.
- Telegram UI остается единственной пользовательской поверхностью сценария.

## Main Flow

1. Пользователь открывает `Корзина` и запускает оформление заказа.
2. Бот запрашивает номер телефона для связи.
3. Пользователь отправляет телефон.
4. Бот запрашивает адрес доставки.
5. Пользователь отправляет адрес.
6. Бот показывает сводку заказа: состав корзины, сумму, телефон, адрес и просит подтвердить создание заказа.
7. Пользователь подтверждает заказ.
8. Система создает заказ с уникальным номером, сохраняет контактные данные, адрес, позиции и итоговую сумму.
9. Бот очищает корзину только после успешного создания заказа и показывает пользователю номер созданного заказа.

## Alternate Flows / Exceptions

- `ALT-01` Если корзина пуста, бот не начинает checkout и показывает `Корзина пуста.`
- `ALT-02` Если у пользователя уже сохранен телефон, бот использует его как значение по умолчанию, но позволяет отправить другой номер.
- `ALT-03` Если пользователь отменяет checkout до подтверждения, заказ не создается, а корзина сохраняется без изменений.
- `EX-01` Если пользователь отправляет пустой или слишком короткий адрес, бот повторно запрашивает корректный адрес.
- `EX-02` Если создание заказа завершается инфраструктурной ошибкой, бот показывает безопасное сообщение об ошибке, не очищает корзину и не сообщает фиктивный номер заказа.

## Postconditions

- После успешного сценария в PostgreSQL сохранен заказ с уникальным номером и составом позиций на момент подтверждения.
- После успешного создания заказа корзина пользователя очищена.
- После неуспешного или отмененного checkout корзина и ее позиции остаются без изменений.

## Business Rules

- `BR-01` Checkout запускается только из непустой корзины.
- `BR-02` До подтверждения заказа корзина не очищается и не теряет позиции.
- `BR-03` Контактные данные и адрес, подтвержденные пользователем, сохраняются в заказе как снапшот и не зависят от будущих изменений профиля.
- `BR-04` Номер заказа должен быть устойчивым и пригодным для показа пользователю.
- `BR-05` Один пользовательский action приводит к одному итоговому ответу: запросу следующего поля, сводке подтверждения, сообщению об успехе или сообщению об ошибке.
