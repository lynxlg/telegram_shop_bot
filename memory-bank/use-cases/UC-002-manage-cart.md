# UC-002: Manage Cart

## Goal

Покупатель собирает корзину внутри Telegram-бота, меняет ее состав и возвращается к тому же сохраненному состоянию после новой сессии.

## Primary Actor

Покупатель интернет-магазина.

## Trigger

Пользователь нажимает `В корзину` в карточке товара или открывает `Корзина` из главного меню.

## Preconditions

- Пользователь зарегистрирован в таблице `users` через текущий flow `/start`.
- Каталог и карточка товара доступны для выбора товара.
- Источником истины для корзины является PostgreSQL.
- Один пользователь имеет не более одной активной корзины.

## Main Flow

1. Пользователь открывает карточку товара и нажимает `В корзину`.
2. Бот создает корзину при первом добавлении или использует существующую корзину пользователя.
3. Бот создает новую позицию с количеством `1` либо увеличивает количество уже существующей позиции того же товара.
4. Пользователь открывает `Корзина`.
5. Бот показывает список позиций с `name`, `price`, `quantity`, стоимостью позиции и итоговой суммой.
6. Пользователь увеличивает, уменьшает или удаляет позицию.
7. Бот обновляет текущий экран корзины.
8. После повторного открытия бота пользователь видит тот же состав корзины, сохраненный в PostgreSQL.

## Alternate Flows / Exceptions

- `ALT-01` Если корзина не содержит позиций, бот показывает `Корзина пуста.`
- `ALT-02` Если пользователь уменьшает количество позиции со значения `1`, позиция удаляется.
- `EX-01` Если товар не существует при добавлении, бот показывает `Товар не найден.`
- `EX-02` Если позиция корзины не существует при изменении или удалении, бот показывает `Позиция корзины не найдена.`
- `EX-03` Если чтение или изменение корзины завершается инфраструктурной ошибкой, бот показывает `Не удалось обновить корзину. Попробуйте позже.`

## Postconditions

- После успешного сценария корзина пользователя сохранена в PostgreSQL и отражает последнее подтвержденное действие.
- Количество любой позиции остается целым числом не меньше `1`; при переходе ниже `1` позиция удаляется.
- Ошибка не очищает корзину автоматически и не показывает пользователю технические детали.

## Business Rules

- `BR-01` В рамках одной корзины не может существовать две разные позиции для одного и того же товара.
- `BR-02` Повторное добавление того же товара увеличивает количество существующей позиции на `1`.
- `BR-03` Для одного пользовательского action бот показывает один итоговый результат: подтверждение добавления, обновленную корзину или сообщение об ошибке.
- `BR-04` Повторная команда `/start`, повторный вход пользователя и рестарт процесса бота не должны очищать корзину автоматически.
- `BR-05` Добавление в корзину не должно ломать навигацию каталога и возврат `Назад` из карточки товара.
