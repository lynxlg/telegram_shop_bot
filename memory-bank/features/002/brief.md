# Brief: Картинки в карточке товара каталога

## Проблема

Сейчас в карточках товаров каталога Telegram-бота отсутствуют изображения: пользователь видит только название, цену, описание и характеристики. Из-за этого он не может визуально оценить товар внутри бота до перехода к следующему шагу выбора.

## Для кого решаем

- Основной пользователь: покупатель интернет-магазина, который выбирает товар внутри Telegram-бота и хочет сразу видеть его изображение.
- Стейкхолдер: владелец магазина / менеджер продаж

## Контекст

В проекте уже реализован read-only каталог: пользователь может открыть раздел, выбрать товар и посмотреть его карточку. В текущем MVP карточка товара является конечной точкой сценария выбора внутри каталога. Без изображения пользователь не может визуально оценить товар внутри бота, поэтому сценарий просмотра каталога остаётся неполным именно на этом этапе развития продукта.

## Желаемый результат

Пользователь может самостоятельно:

- открыть товар в каталоге;
- увидеть изображение товара прямо в карточке;
- получить текстовую информацию о товаре вместе с визуальным представлением;
- вернуться из карточки товара на предыдущий экран каталога, из которого он открыл товар.

## Критерий готовности

Задача считается решённой, если пользователь без участия менеджера может пройти путь:

1. открыть каталог;
2. перейти в нужную категорию;
3. открыть товар;
4. увидеть карточку товара с изображением;
5. получить название, цену, описание и характеристики товара вместе с картинкой, достаточные для выбора товара и перехода к следующему шагу заказа.
