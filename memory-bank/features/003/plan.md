# План реализации: Корзина товаров в Telegram-боте

### 1. Добавить ORM-модели корзины

- Создать файл `app/models/cart.py` с моделью `Cart`:
  - `id: BigInteger`
  - `user_id: BigInteger`
  - связь с `User`
  - связь `items` с `CartItem`
- Создать файл `app/models/cart_item.py` с моделью `CartItem`:
  - `id: BigInteger`
  - `cart_id: BigInteger`
  - `product_id: BigInteger`
  - `quantity: int`
  - связи с `Cart` и `Product`
- В модели [user.py](/home/lynx/telegram_shop_bot/app/models/user.py) добавить связь с корзиной.
- В модели [product.py](/home/lynx/telegram_shop_bot/app/models/product.py) добавить связь с позициями корзины.
- На уровне ORM и схемы зафиксировать инварианты:
  - одна корзина на пользователя;
  - одна позиция на товар в рамках корзины;
  - `quantity >= 1`.

### 2. Подключить новые модели к metadata приложения и Alembic

- Обновить [database.py](/home/lynx/telegram_shop_bot/app/models/database.py):
  - импортировать `app.models.cart`;
  - импортировать `app.models.cart_item`;
  - добавить их в `init_db()` рядом с существующими моделями.
- Обновить [alembic/env.py](/home/lynx/telegram_shop_bot/alembic/env.py):
  - добавить импорты `Cart` и `CartItem`, чтобы новые таблицы входили в `target_metadata`.

### 3. Создать Alembic-миграцию корзины

- Создать новую миграцию в `alembic/versions/`.
- Добавить таблицу `carts`:
  - `id`
  - `user_id`
  - `created_at`, если это поле решено хранить в модели
  - `UNIQUE (user_id)`
- Добавить таблицу `cart_items`:
  - `id`
  - `cart_id`
  - `product_id`
  - `quantity`
  - `UNIQUE (cart_id, product_id)`
  - `CHECK (quantity >= 1)`
- Добавить индексы и внешние ключи на `users`, `products`, `carts`.

### 4. Реализовать сервисный слой корзины

- Создать файл `app/services/cart.py`.
- Добавить async-функции:
  - `get_or_create_cart_by_telegram_id(session, telegram_id)`
  - `add_product_to_cart(session, telegram_id, product_id)`
  - `get_cart_by_telegram_id(session, telegram_id)`
  - `increase_cart_item_quantity(session, cart_item_id)`
  - `decrease_cart_item_quantity(session, cart_item_id)`
  - `remove_cart_item(session, cart_item_id)`
- Внутри сервиса:
  - искать пользователя по `users.telegram_id`;
  - создавать корзину при первом добавлении;
  - увеличивать количество при повторном добавлении того же товара;
  - удалять позицию при уменьшении количества с `1`;
  - делать `commit` и `rollback` в явных точках изменения данных;
  - логировать ошибки БД.

## Критерий готовности

Фича считается готовой, когда:

- пользователь может добавить товар в корзину из карточки товара;
- пользователь может открыть корзину из главного меню;
- состав корзины хранится в PostgreSQL и сохраняется между сессиями;
- количество позиций можно увеличивать и уменьшать;
- позицию можно удалить отдельной кнопкой;
- суммы по позициям и общая сумма пересчитываются корректно;
- кнопка `Назад` в каталоге не сломана;
- миграции, тесты и ручные сценарии проходят.
