# Brief: Корзина товаров в Telegram-боте

## Проблема

Сейчас пользователь Telegram-бота интернет-магазина может только просматривать каталог и карточки товаров, но не может сохранить выбранные позиции для последующего оформления заказа. Из-за этого выбор товара обрывается на просмотре карточки: пользователь не может добавить товар в корзину, изменить количество или удалить позицию, а при закрытии бота его выбор полностью теряется.

## Для кого решаем

- Основной пользователь: покупатель интернет-магазина, который хочет собрать заказ внутри Telegram-бота и вернуться к нему позже без потери выбранных товаров.
- Стейкхолдер: владелец магазина / менеджер продаж

## Контекст

В проекте уже реализован базовый каталог товаров с карточкой товара, поэтому следующим обязательным шагом после просмотра товара становится сохранение выбора пользователя. Корзина нужна как переход между каталогом и будущим оформлением заказа: без неё пользователь не может собрать несколько товаров, скорректировать состав заказа и продолжить покупку после повторного открытия бота. Дополнительное требование на этом этапе: выбранные товары должны сохраняться между сессиями, чтобы пользователь не терял корзину после закрытия бота.

## Желаемый результат

Пользователь может самостоятельно:

- добавить товар в корзину из карточки товара;
- открыть корзину и увидеть выбранные позиции;
- изменить количество товара в корзине;
- удалить товар из корзины;
- закрыть бота, вернуться позже и продолжить работу с сохранённой корзиной.

## Критерий готовности

Задача считается решённой, если пользователь без участия менеджера может пройти путь:

1. открыть каталог;
2. перейти в карточку товара;
3. добавить товар в корзину;
4. открыть корзину и увидеть выбранную позицию;
5. изменить количество товара или удалить позицию;
6. закрыть бота и открыть его снова;
7. получить корзину с тем же составом товаров и тем же количеством по каждой позиции, которое было до закрытия бота.
