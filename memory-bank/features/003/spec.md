# Spec: Корзина товаров в Telegram-боте

## Ссылка на brief

- Источник: `memory-bank/features/003/brief.md`

## Цель

Добавить в Telegram-бот корзину, чтобы пользователь мог:

1. добавить товар в корзину из карточки товара;
2. открыть корзину и увидеть позиции;
3. изменить количество товара в корзине;
4. удалить товар из корзины;
5. закрыть бота и позже продолжить работу с той же сохраненной корзиной.

## Scope

Входит в scope:

- кнопка `В корзину` в карточке товара;
- кнопка `Корзина` в главном меню;
- хранение корзины пользователя в PostgreSQL;
- сохранение состава корзины между сессиями;
- просмотр списка позиций корзины;
- отображение количества товара по каждой позиции;
- изменение количества товара в корзине;
- удаление позиции из корзины;
- пересчет итогового состава корзины после изменения количества или удаления позиции;
- обработка состояний `success`, `empty`, `error`, `loading`.

Не входит в scope:

- оформление заказа;
- сбор контактных данных;
- подтверждение заказа;
- генерация номера заказа;
- онлайн-оплата;
- статусы оплаты и заказа;
- доставка;
- админское управление корзинами;
- промокоды, скидки и резервирование остатков;
- поиск и фильтры;
- массовые операции над корзиной, кроме просмотра, изменения количества и удаления позиции.

## Пользователи и стейкхолдеры

- Основной пользователь: покупатель интернет-магазина.
- Стейкхолдер: владелец магазина или менеджер продаж.

## Предпосылки

- В системе уже реализованы каталог, карточка товара и пользовательская регистрация по `/start`.
- Пользователь может дойти до карточки товара через текущую навигацию каталога.
- Каждый товар имеет уникальный идентификатор, `name` и `price`.
- Корзина привязана к пользователю из таблицы `users`.
- Корзина должна переживать завершение сессии и повторный запуск бота.
- В рамках фичи один пользователь имеет одну актуальную корзину.
- В рамках фичи товар в корзине хранится как отдельная позиция с количеством.
- Каталог остается источником данных о товаре.

## Термины

- Корзина: сохраненный набор выбранных пользователем товаров с количеством по каждой позиции.
- Позиция корзины: связь между корзиной пользователя и конкретным товаром с полем количества.
- Количество: целое положительное число единиц товара в одной позиции корзины.
- Пустая корзина: корзина без позиций.
- Сохранение между сессиями: состояние корзины остается в БД и доступно после повторного открытия бота.

## Пользовательский сценарий

1. Пользователь открывает каталог и переходит в карточку товара.
2. Пользователь нажимает `В корзину`.
3. Бот сохраняет товар в корзине текущего пользователя.
4. Если товар уже есть в корзине, бот увеличивает количество существующей позиции.
5. Пользователь открывает `Корзина`.
6. Бот показывает список позиций корзины с количеством по каждой позиции.
7. Пользователь изменяет количество товара в корзине или удаляет позицию.
8. Бот показывает обновленное состояние корзины.
9. Пользователь закрывает бота и возвращается.
10. Бот по кнопке `Корзина` показывает тот же сохраненный состав товаров и количества.

## Состояния интерфейса

### Success

- Товар успешно добавлен в корзину.
- Корзина успешно открыта.
- Количество позиции успешно изменено.
- Позиция успешно удалена.

### Empty

- В корзине нет ни одной позиции.

### Error

- Товар не найден при попытке добавить его в корзину.
- Позиция корзины не найдена.
- Ошибка чтения или изменения данных корзины в БД.

### Loading

- Во время добавления товара и изменения корзины бот не должен показывать пользователю промежуточные или частично обновленные состояния корзины.
- Для одного действия пользователя бот должен показывать один итоговый пользовательский результат: сообщение `Товар добавлен в корзину.`, обновленную корзину или сообщение об ошибке.

## Функциональные требования

### 1. Добавление товара в корзину

- В карточке товара должна быть кнопка `В корзину`.
- При нажатии `В корзину` бот должен сохранить товар в корзине текущего пользователя.
- Если у пользователя еще нет корзины, бот должен создать ее автоматически при первом добавлении товара.
- Если товара еще нет в корзине, бот должен создать новую позицию с количеством `1`.
- Если товар уже есть в корзине, бот должен увеличить количество существующей позиции на `1`.
- Если товар не существует, бот должен показать сообщение `Товар не найден.`

### 2. Открытие корзины

- В главном меню бота должна быть кнопка `Корзина`.
- При выборе `Корзина` бот должен показать все позиции корзины текущего пользователя, сохраненные в БД на момент обработки запроса.
- Для каждой позиции в корзине бот должен показать:
- `name`;
- `price`;
- `quantity`;
- стоимость позиции, рассчитанную как `price * quantity`.
- Бот должен показывать итоговую сумму корзины.
- Если корзина пуста, бот должен показать сообщение `Корзина пуста.`

### 3. Изменение количества

- Для каждой позиции корзины пользователь должен иметь возможность увеличить количество на `1`.
- Для каждой позиции корзины пользователь должен иметь возможность уменьшить количество на `1`, пока количество больше `1`.
- Количество позиции не может быть меньше `1`.
- После изменения количества бот должен показать обновленное состояние корзины.
- Если пользователь пытается уменьшить количество позиции `1`, бот должен удалить позицию из корзины или предоставить отдельное действие удаления.
- В рамках этой фичи допускается один фиксированный вариант поведения:
- при уменьшении количества с `1` позиция удаляется из корзины.

### 4. Удаление позиции

- Для каждой позиции корзины бот должен показывать inline-кнопку `Удалить`.
- После удаления позиции бот должен показать обновленное состояние корзины.
- Если после удаления корзина стала пустой, бот должен показать сообщение `Корзина пуста.`

### 5. Сохранение между сессиями

- Корзина и ее позиции должны храниться в PostgreSQL.
- Повторный запуск бота, повторная команда `/start` или повторный вход пользователя не должны очищать корзину автоматически.
- После повторного открытия бота пользователь должен видеть ту же корзину с теми же товарами и количествами, которые были сохранены ранее.

### 6. Навигация

- Пользователь должен иметь возможность открыть корзину из главного меню.
- Добавление кнопки `В корзину` не должно ломать текущую навигацию каталога и кнопку `Назад` в карточке товара.
- Действия внутри корзины должны обновлять текущий экран корзины без необходимости заново проходить каталог.

### 7. Ошибки

- Если товар не существует при добавлении в корзину, бот должен показать `Товар не найден.`
- Если позиция корзины не существует при изменении количества или удалении, бот должен показать `Позиция корзины не найдена.`
- Если запрос к БД завершается ошибкой при операциях корзины, бот должен показать сообщение `Не удалось обновить корзину. Попробуйте позже.`
- Во всех сценариях ошибок бот не должен отправлять traceback пользователю.
- Во всех сценариях ошибок приложение должно записывать ошибку в лог.

## Нефункциональные ограничения

- Реализация должна использовать текущий стек проекта: `Aiogram`, `SQLAlchemy async`, `PostgreSQL`.
- Все операции ввода-вывода должны быть асинхронными.
- Доступ к БД должен проходить через текущий session-based подход приложения.
- Сохранение корзины должно опираться на постоянное хранилище, а не на in-memory state.
- Реализация должна затрагивать не более 3 зон системы:
- модели корзины и миграции;
- сервисы работы с корзиной;
- handler/UI корзины и карточки товара.

## Инварианты

- У одного пользователя в каждый момент времени есть не более одной активной корзины.
- В одной корзине не может быть двух разных позиций для одного и того же товара.
- Количество каждой позиции корзины всегда целое и не меньше `1`.
- После успешного добавления товара пользователь видит сообщение `Товар добавлен в корзину.`, а после изменения корзины видит обновленное состояние корзины.
- Корзина не очищается автоматически из-за повторного запуска бота или повторного входа пользователя.
- Ошибка корзины всегда приводит к пользовательскому сообщению без технических деталей и к записи ошибки в лог.

## Acceptance Criteria

1. Если пользователь открывает карточку товара и нажимает `В корзину`, товар сохраняется в корзине текущего пользователя.
2. Если пользователь добавляет товар, которого еще нет в корзине, в корзине создается позиция с количеством `1`.
3. Если пользователь повторно добавляет тот же товар, количество существующей позиции увеличивается на `1`.
4. Если пользователь открывает `Корзина`, бот показывает все сохраненные позиции с `name`, `price`, `quantity`, стоимостью позиции и итоговой суммой корзины.
5. Если корзина не содержит позиций, бот показывает `Корзина пуста.`
6. Если пользователь увеличивает количество позиции, бот показывает обновленную корзину с увеличенным количеством и пересчитанной суммой.
7. Если пользователь уменьшает количество позиции больше `1`, бот показывает обновленную корзину с уменьшенным количеством и пересчитанной суммой.
8. Если пользователь уменьшает количество позиции `1`, позиция удаляется из корзины, и бот показывает обновленную корзину или `Корзина пуста.`, если это была последняя позиция.
9. Если пользователь нажимает inline-кнопку `Удалить` у позиции корзины, позиция исчезает из корзины, а бот показывает обновленное состояние корзины.
10. Если пользователь закрывает бота и позже снова открывает его, при открытии `Корзина` бот показывает тот же состав товаров и те же количества, которые были сохранены до закрытия бота.
11. Если товар не существует при добавлении в корзину, бот показывает `Товар не найден.`
12. Если позиция корзины не существует при изменении количества или удалении, бот показывает `Позиция корзины не найдена.`
13. Если любой запрос к БД для операций корзины завершается ошибкой, бот показывает `Не удалось обновить корзину. Попробуйте позже.` и пишет ошибку в лог.
14. Если пользователь открывает карточку товара, нажимает `Назад`, бот возвращает его на предыдущий экран каталога; добавление кнопки `В корзину` не меняет это поведение.
15. Для одного действия пользователя бот показывает не более одного итогового пользовательского ответа.

## Критерии завершения

Фича считается готовой, когда пользователь может пройти путь из brief:

1. открыть каталог;
2. перейти в карточку товара;
3. добавить товар в корзину;
4. открыть корзину и увидеть выбранную позицию;
5. изменить количество товара или удалить позицию;
6. закрыть бота и открыть его снова;
7. получить корзину с тем же составом товаров и тем же количеством по каждой позиции, которое было до закрытия бота.
