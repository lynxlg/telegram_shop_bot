# Spec: Картинки в карточке товара каталога

## Ссылка на brief

- Источник: `memory-bank/features/legacy/002/brief.md`

## Цель

Добавить в Telegram-бот отображение одного изображения в карточке товара, чтобы пользователь мог без участия менеджера:

1. открыть товар в каталоге;
2. увидеть изображение товара в карточке;
3. получить `name`, `price`, `description` и `attributes` вместе с изображением;
4. вернуться на предыдущий экран каталога, из которого был открыт товар.

## Scope

Входит в scope:

- хранение у товара одного поля `image_url`;
- чтение `image_url` при открытии карточки товара;
- показ карточки товара с изображением, если `image_url` задан;
- показ карточки товара без изображения, если `image_url` отсутствует;
- сохранение текущей логики кнопки `Назад` для карточки товара;
- обработка ошибок чтения данных товара и ошибок показа карточки.

Не входит в scope:

- загрузка изображений через Telegram админом;
- поддержка `file_id`, локальных файлов и других форматов хранения изображения, кроме `image_url`;
- множественные изображения товара;
- галерея изображений;
- изменение сценариев списка категорий;
- изменение сценариев списка товаров;
- корзина;
- оформление заказа;
- оплата;
- поиск и фильтры.

## Пользователи и стейкхолдеры

- Основной пользователь: покупатель интернет-магазина.
- Стейкхолдер: владелец магазина или менеджер продаж.

## Предпосылки

- В системе уже реализован read-only каталог с категориями, товарами и карточкой товара.
- Пользователь может дойти до карточки товара через текущую навигацию каталога.
- Для каждого отображаемого товара обязательны поля `name` и `price`.
- Поля `description` и `attributes` могут отсутствовать.
- У товара может быть поле `image_url` типа `str`, содержащее URL изображения.
- Если `image_url` пустой или `NULL`, товар считается товаром без изображения.
- Каталог работает только в режиме чтения данных.

## Термины

- Карточка товара: экран каталога, на котором пользователь видит данные выбранного товара.
- Изображение товара: изображение по значению поля `image_url` текущего товара.
- Товар без изображения: товар, у которого `image_url` отсутствует, пустой или равен `NULL`.
- Предыдущий экран каталога: тот экран списка товаров или категорий, с которого пользователь открыл карточку товара.
- Fallback-значение: фиксированный текст, который бот показывает вместо отсутствующего поля.

## Пользовательский сценарий

1. Пользователь открывает каталог и доходит до нужного товара.
2. Пользователь выбирает товар в списке.
3. Если у товара задан `image_url`, бот показывает карточку товара с изображением и текстом.
4. Если у товара `image_url` отсутствует, бот показывает карточку товара без изображения, но с текстом.
5. Пользователь нажимает `Назад`.
6. Бот возвращает пользователя на тот экран каталога, с которого был открыт товар.

## Состояния интерфейса

### Success

- Карточка товара с изображением успешно показана.
- Карточка товара без изображения успешно показана.

### Empty

- У товара отсутствует `image_url`, но карточка товара успешно показана без изображения.

### Error

- Товар не найден.
- Ошибка чтения данных товара из БД.
- Ошибка показа карточки товара пользователю.

### Loading

- Во время открытия карточки товара бот не должен показывать пользователю промежуточные или частично заполненные сообщения.
- Для одного действия пользователя бот должен показать один итоговый результат: карточку товара с изображением, карточку товара без изображения или сообщение об ошибке.

## Функциональные требования

### 1. Данные изображения товара

- У товара должно быть не более одного поля изображения: `image_url`.
- В рамках этой фичи только поле `image_url` используется для показа изображения в карточке товара.
- Если `image_url` отсутствует, пустой или равен `NULL`, товар считается товаром без изображения.

### 2. Отображение карточки товара

- При открытии товара бот должен показать карточку товара.
- Если у товара задан `image_url`, бот должен показать карточку товара с изображением.
- Если у товара нет `image_url`, бот должен показать карточку товара без изображения.
- Карточка товара должна содержать:
- `name`;
- `price`;
- `description` или текст `Описание отсутствует.`;
- `attributes` списком или текст `Характеристики не указаны.`
- Если у товара несколько характеристик, каждая характеристика должна отображаться отдельной строкой в формате `Название: значение`.

### 3. Поведение при отсутствии изображения

- Отсутствие `image_url` не должно блокировать просмотр `name`, `price`, `description` и `attributes`.
- Для товара без изображения бот не должен показывать пользователю технические ошибки или внутренние детали хранения данных.

### 4. Навигация

- После открытия карточки товара пользователь должен иметь возможность нажать `Назад`.
- Кнопка `Назад` должна возвращать пользователя на тот экран каталога, с которого был открыт товар.
- Добавление `image_url` в карточку товара не должно менять текущую логику переходов по категориям и товарам.

### 5. Ошибки

- Если запрошенный товар не существует, бот должен показать сообщение `Товар не найден.`
- Если запрос к БД завершается ошибкой при открытии товара, бот должен показать сообщение `Не удалось загрузить каталог. Попробуйте позже.`
- Если карточку товара с изображением невозможно показать пользователю по любой причине, бот должен показать сообщение `Не удалось загрузить каталог. Попробуйте позже.`
- Если карточку товара с изображением невозможно показать пользователю, бот не должен переключаться на fallback-сценарий показа карточки без изображения.
- Во всех сценариях ошибок бот не должен отправлять traceback пользователю.
- Во всех сценариях ошибок приложение должно записывать ошибку в лог.

## Нефункциональные ограничения

- Реализация должна использовать текущий стек проекта: `Aiogram`, `SQLAlchemy async`, `PostgreSQL`.
- Все операции ввода-вывода должны быть асинхронными.
- Доступ к БД должен проходить через текущий session-based подход приложения.
- Реализация должна затрагивать не более 3 модулей/слоёв:
- модель данных товара;
- сервис чтения карточки товара;
- handler/UI карточки товара.

## Инварианты

- Пользователь всегда получает один итоговый результат открытия товара: карточку с изображением, карточку без изображения или сообщение об ошибке.
- Карточка товара всегда содержит `name` и `price`.
- Отсутствие `image_url` не делает товар недоступным для просмотра.
- Ошибка показа карточки товара с изображением всегда приводит к сообщению `Не удалось загрузить каталог. Попробуйте позже.`
- Навигация `Назад` после открытия карточки товара всегда возвращает пользователя на предыдущий экран каталога.
- Сценарий просмотра карточки товара не создает, не изменяет и не удаляет данные каталога.

## Acceptance Criteria

1. Если пользователь открывает товар с непустым `image_url`, бот показывает карточку товара с изображением.
2. Если пользователь открывает товар с пустым `image_url` или `NULL`, бот показывает карточку товара без изображения, но с `name`, `price`, `description` и `attributes` либо их fallback-значениями.
3. Если у товара нет описания, бот показывает `Описание отсутствует.`
4. Если у товара нет характеристик, бот показывает `Характеристики не указаны.`
5. Если пользователь нажимает `Назад` из карточки товара, бот возвращает его на тот экран каталога, с которого был открыт товар.
6. Если товар не существует, бот показывает `Товар не найден.`
7. Если чтение данных товара из БД завершается ошибкой, бот показывает `Не удалось загрузить каталог. Попробуйте позже.` и пишет ошибку в лог.
8. Если показ карточки товара с изображением завершается ошибкой, бот показывает `Не удалось загрузить каталог. Попробуйте позже.` и пишет ошибку в лог.
9. Для одного действия пользователя бот показывает не более одного итогового пользовательского ответа.

## Критерии завершения

Фича считается готовой, когда пользователь может пройти путь из brief:

1. открыть каталог;
2. перейти в нужную категорию;
3. открыть товар;
4. увидеть карточку товара с изображением, если у товара задан `image_url`;
5. получить `name`, `price`, `description` и `attributes`, а при отсутствии `image_url` увидеть карточку товара без изображения.
