# Spec: Каталог товаров в Telegram-боте

## Ссылка на brief

- Источник: `memory-bank/features/legacy/001/brief.md`

## Цель

Добавить в Telegram-бот сценарий просмотра каталога, чтобы пользователь мог без участия менеджера:

1. открыть каталог;
2. перейти в выбранную категорию или подкатегорию;
3. открыть карточку товара;
4. получить `name`, `price`, `description` и `attributes` либо их fallback-значения.

## Scope

Входит в scope:

- кнопка `Каталог` в главном меню;
- просмотр списка корневых категорий;
- переход по иерархии категорий;
- просмотр списка товаров внутри листовой категории;
- открытие карточки товара;
- отображение названия, цены, описания и характеристик товара;
- обработка состояний `success`, `empty`, `error`, `loading`;
- навигация `Назад` внутри каталога.

Не входит в scope:

- корзина;
- оформление заказа;
- оплата;
- поиск;
- фильтры;
- отзывы;
- админское управление каталогом;
- создание, редактирование и удаление товаров и категорий через Telegram.

## Пользователи и стейкхолдеры

- Основной пользователь: покупатель интернет-магазина.
- Стейкхолдер: владелец магазина или менеджер продаж.

## Предпосылки

- В системе есть сохраненные категории и товары.
- Категория может иметь дочерние категории.
- Листовая категория не имеет дочерних категорий и может содержать товары.
- Товар привязан ровно к одной категории.
- Для каждого отображаемого товара обязательны поля `name` и `price`.
- Поля `description` и `attributes` могут отсутствовать.
- Каталог работает только в режиме чтения данных.

## Термины

- Корневая категория: категория без родительской категории.
- Дочерняя категория: категория, у которой `parent_id` указывает на текущую категорию.
- Листовая категория: категория без дочерних категорий.
- Активный товар: товар, разрешенный к показу в каталоге. Правила определения активности задаются моделью данных и не меняются в рамках этой фичи.
- Характеристики товара: список пар `название: значение`, сохраненный в БД для карточки товара.
- Fallback-значение: фиксированный текст, который бот показывает вместо отсутствующего поля.

## Пользовательский сценарий

1. Пользователь нажимает кнопку `Каталог`.
2. Бот показывает список корневых категорий.
3. Пользователь выбирает категорию.
4. Если у категории есть дочерние категории, бот показывает список дочерних категорий.
5. Если у категории нет дочерних категорий, бот показывает список товаров этой категории.
6. Пользователь выбирает товар.
7. Бот показывает карточку товара.
8. Пользователь может нажать `Назад` и вернуться на предыдущий экран каталога.

## Состояния интерфейса

### Success

- Список корневых категорий успешно загружен.
- Список дочерних категорий успешно загружен.
- Список товаров успешно загружен.
- Карточка товара успешно загружена.

### Empty

- В каталоге нет корневых категорий.
- В листовой категории нет товаров.

### Error

- Категория не найдена.
- Товар не найден.
- Ошибка чтения данных из БД.

### Loading

- Во время обработки команды или callback бот не должен отправлять пользователю промежуточные сообщения с неполными данными.
- Для одного действия пользователя бот должен отправлять только один итоговый пользовательский ответ: список категорий, список товаров, карточку товара или сообщение об ошибке.

## Функциональные требования

### 1. Вход в каталог

- В главном меню бота должна быть кнопка `Каталог`.
- При выборе `Каталог` бот должен показать список корневых категорий.
- Если корневых категорий нет, бот должен показать сообщение `Каталог пока пуст.`

### 2. Навигация по категориям

- Каталог должен поддерживать иерархию минимум в 2 уровня.
- Для выбранной категории бот должен показывать только дочерние категории текущего уровня.
- Если у выбранной категории есть дочерние категории, бот не должен показывать товары этой категории в том же ответе.
- На каждом экране каталога, кроме корневого списка, должна быть кнопка `Назад`.
- При нажатии `Назад` бот должен возвращать пользователя ровно на один уровень выше.

### 3. Список товаров

- Если у выбранной категории нет дочерних категорий, бот должен показать список активных товаров этой категории.
- Для каждого товара в списке бот должен показать:
- `name`;
- `price`.
- Если в листовой категории нет товаров, бот должен показать сообщение `В этом разделе пока нет товаров.`

### 4. Карточка товара

- При выборе товара бот должен показать карточку товара.
- Карточка товара должна содержать:
- `name`;
- `price`;
- `description` или текст `Описание отсутствует.`;
- `attributes` списком или текст `Характеристики не указаны.`
- Если у товара несколько характеристик, каждая характеристика должна отображаться отдельной строкой в формате `Название: значение`.

### 5. Ошибки

- Если запрошенная категория не существует, бот должен показать сообщение `Раздел не найден.`
- Если запрошенный товар не существует, бот должен показать сообщение `Товар не найден.`
- Если запрос к БД завершается любой ошибкой при открытии каталога, категории или товара, бот должен показать сообщение `Не удалось загрузить каталог. Попробуйте позже.`
- Во всех сценариях ошибок бот не должен отправлять traceback пользователю.
- Во всех сценариях ошибок приложение должно записывать ошибку в лог.

## Нефункциональные ограничения

- Реализация должна использовать текущий стек проекта: `Aiogram`, `SQLAlchemy async`, `PostgreSQL`.
- Все операции ввода-вывода должны быть асинхронными.
- Доступ к БД должен проходить через текущий session-based подход приложения.
- Реализация этой фичи должна затрагивать не более 3 зон системы:
- пользовательские handlers;
- модели и запросы каталога;
- клавиатуры и навигацию бота.

## Инварианты

- Пользователь всегда видит один из четырех результатов текущего шага: список категорий, список товаров, карточку товара или сообщение об ошибке.
- Пользователь не может открыть карточку несуществующего товара без сообщения `Товар не найден.`
- Карточка товара всегда содержит `name` и `price`.
- Навигация `Назад` всегда переводит пользователя на один предыдущий экран внутри каталога.
- Сценарий каталога не создает, не изменяет и не удаляет данные.

## Acceptance Criteria

1. Если пользователь нажимает `Каталог`, бот показывает список корневых категорий.
2. Если корневых категорий нет, бот показывает `Каталог пока пуст.`
3. Если пользователь выбирает категорию с дочерними категориями, бот показывает только дочерние категории этой категории.
4. Если пользователь выбирает листовую категорию с товарами, бот показывает список товаров этой категории.
5. Если пользователь выбирает листовую категорию без товаров, бот показывает `В этом разделе пока нет товаров.`
6. Если пользователь выбирает товар с описанием и характеристиками, бот показывает `name`, `price`, `description` и список `attributes`.
7. Если у товара нет описания, бот показывает `Описание отсутствует.`
8. Если у товара нет характеристик, бот показывает `Характеристики не указаны.`
9. Если пользователь нажимает `Назад`, бот возвращает его ровно на один уровень выше.
10. Если категория не существует, бот показывает `Раздел не найден.`
11. Если товар не существует, бот показывает `Товар не найден.`
12. Если запрос к БД завершается ошибкой, бот показывает `Не удалось загрузить каталог. Попробуйте позже.` и пишет ошибку в лог.
13. Для одного действия пользователя бот отправляет не более одного итогового пользовательского сообщения.

## Критерии завершения

Фича считается готовой, когда пользователь может пройти путь из brief:

1. открыть каталог;
2. перейти в нужный раздел ассортимента;
3. открыть товар;
4. получить `name`, `price`, `description` и `attributes` либо их fallback-значения.
